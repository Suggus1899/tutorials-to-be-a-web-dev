# Día 2: Branching Strategies

En este día aprenderás estrategias de branching.

## Contenido
- Git Flow
- GitHub Flow
- Trunk-based development
- Cuándo usar cada una

## Tareas

### Tarea 1: Git Flow
Entiende las ramas:
- main/master: producción
- develop: integración
- feature/*: nuevas funcionalidades
- release/*: preparación de release
- hotfix/*: arreglos urgentes

### Tarea 2: GitHub Flow
Flujo simplificado:
- main siempre deployable
- Feature branches del main
- Pull requests para merge
- Deploy después de merge

### Tarea 3: Practica Git Flow
Simula un flujo completo:
1. Crea feature branch
2. Desarrolla y commit
3. Merge a develop
4. Crea release branch
5. Merge a main

## Ejercicios Adicionales
- Investiga Trunk-based development
- Decide qué estrategia usar en tus proyectos
