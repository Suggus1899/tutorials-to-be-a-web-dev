# Día 1: Introducción al Testing

En este día aprenderás los fundamentos del testing de software.

## Contenido
- Por qué hacer testing
- Tipos de tests (unitarios, integración, E2E)
- Pirámide de testing
- TDD vs testing tradicional

## Tareas

### Tarea 1: Conceptos de testing
Estudia y documenta:
- Beneficios del testing automático
- Diferencia entre test manual y automático
- Qué es un test case

### Tarea 2: Tipos de tests
Entiende cada tipo:
- Unitarios: prueban funciones aisladas
- Integración: prueban componentes juntos
- E2E: prueban el sistema completo

### Tarea 3: La pirámide de testing
Analiza:
- Muchos tests unitarios (base)
- Menos tests de integración (medio)
- Pocos tests E2E (cima)
- Por qué esta proporción

## Ejercicios Adicionales
- Investiga TDD (Test Driven Development)
- Lee sobre code coverage
